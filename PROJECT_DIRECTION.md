# Project Direction

## Positioning

This project is not a normal Travellers Rest Wiki, and it is not a database-only reference site.

The project direction is:

**Travellers Rest Tavern Planner & Strategy Guide**

The site should help players decide what to do next, what to cook, which ingredients to stockpile, how to prepare for trends, and how to run a better tavern.

## Core Difference

Wiki-style sites answer:

- What is this?
- Where does this item appear?
- What are the raw stats?

This site should answer:

- What should I do right now?
- Why should I do it?
- What are my alternatives?
- What should I avoid?
- Where should I go next?

The database exists to support decisions. It is not the main product.

## Primary Entrances

Future navigation, homepage content, and major feature work should prioritize:

- What to Do Next
- Menu Planner
- Tavern Progression Guide
- Strategy Guides
- Problem Solver

These entrances should lead players from a problem or goal into a practical recommendation.

## Supporting Database Role

Database pages such as Recipes, Ingredients, Crops, Fish, Drinks, and Stations should remain available, but they are supporting tools for planning and strategy.

They should not become the main selling point of the site. They should help players verify ingredients, stations, categories, and relationships after a planner or guide recommends a route.

## Required Planning Lens

Every new page, feature, or content module should prioritize:

- Player stage
- Player goal
- Player problem
- Decision recommendation
- Next action
- Common mistakes
- Related tools
- Related database links

If a proposed page only lists information without helping a player choose an action, redesign it as a strategy page, planner, checklist, or problem-solving guide.

## Content Direction

The site should feel like a decision assistant and tavern operations planner. It should help players connect farming, cooking, brewing, fishing, staff, rooms, inventory, trends, reputation, and income into a playable route.

The ideal page answers a concrete player question and sends the player to the next useful action.
